# UX-SPEC — Czech Realities Frontend (MVP)

## 1) Цель и контекст

**Продукт:** веб-интерфейс для тренировок к экзамену/тестам по теме `czech_realities`.

**Цель MVP:** дать быстрый, визуально приятный и понятный мобильный опыт прохождения вопросов в 3 режимах с прозрачным прогрессом и результатами.

**Ключевые UX-принципы:**
- Mobile-first (основной сценарий — смартфон, вертикальная ориентация).
- Минимум шагов до старта (1–2 тапа).
- Один фокус на экране (single-task UI).
- Читаемость текста + поддержка иллюстраций.
- Мгновенная обратная связь после ответа.

---

## 2) Информационная архитектура (MVP)

### Основные экраны
1. **Главный экран (Mode Select)**
2. **Экран вопроса (Question Screen)**
3. **Экран результатов/прогресса (Results & Progress)**

### Навигационная модель
- Поток линейный: `Главный → Вопросы → Результаты`
- На каждом экране верхняя шапка с:
  - кнопкой назад (кроме главного),
  - названием режима,
  - индикатором прогресса (на экране вопроса).
- После завершения сессии пользователь попадает на результаты с CTA:
  - «Повторить режим»
  - «На главный»

---

## 3) Главный экран — 3 режима

## 3.1 Содержимое
- Заголовок: **«Czech Realities»**
- Подзаголовок: **«Тренируйся быстро и регулярно»**
- 3 крупные карточки-режима:
  1. **Случайный вопрос**
  2. **По теме**
  3. **Случайный билет**
- Вторичные элементы:
  - мини-блок текущей статистики (например: точность за 7 дней)
  - кнопка «Продолжить последнюю сессию» (если есть незавершённая)

## 3.2 Поведение режимов

### Режим 1: Случайный вопрос
- Стартует один случайный вопрос.
- После ответа:
  - либо возвращение на главный,
  - либо CTA «Следующий случайный» (для короткой цепочки).

### Режим 2: По теме
- Шаг 1: выбор темы (bottom sheet или отдельный список).
- Шаг 2: запуск сессии вопросов выбранной темы.
- Вопросы идут последовательно/случайно в рамках темы (MVP: случайно в теме).

### Режим 3: Случайный билет
- Выбирается случайный билет (набор N вопросов).
- Прохождение билета целиком до экрана результатов.

## 3.3 UX-требования к карточкам режимов
- Вся карточка кликабельна.
- Иконка + заголовок + короткое описание (1 строка).
- Явные hover/press/focus состояния.
- Минимальная высота карточки: 72px (touch-friendly).

## 4) Экран вопроса

## 4.1 Состав экрана
1. **Top bar**
   - Название режима
   - Прогресс (`Вопрос 3/20`)
   - Линейный progress bar

2. **Блок вопроса**
   - Текст вопроса (обязателен)
   - Изображение (опционально)

3. **Блок ответов**
   - Ровно 4 варианта ответа (single choice)
   - Компонент: radio-card (карточка с радиовыбором)

4. **Зона действия**
   - Кнопка `Подтвердить` (disabled до выбора ответа)
   - После ответа: `Следующий вопрос` / `Завершить`

## 4.2 Состояния экрана вопроса
- **Default:** вопрос + 4 невыбранных ответа.
- **Selected:** выбран один вариант, активна кнопка подтверждения.
- **Answered-correct:** правильный ответ подсвечен зелёным, пояснение (если есть).
- **Answered-wrong:** выбранный неверный — красный, правильный — зелёный.
- **Loading:** skeleton для текста/картинки/ответов.
- **Error:** retry-состояние при ошибке загрузки.

## 4.3 Изображения в вопросе
- Соотношение сторон: адаптивно (рекомендуемый контейнер 16:9 / auto-fit).
- `object-fit: contain`, чтобы не терять важные детали.
- Тап по картинке → полноэкранный просмотр (MVP optional, but recommended).
- Обязательный alt-текст из данных вопроса (или fallback).

## 4.4 UX-правила ответов
- После подтверждения ответы блокируются от повторного выбора.
- Обратная связь не более чем через 150ms после действия.
- Для неверного ответа показывать краткий корректный вариант/подсказку (если есть в контенте).

---

## 5) Экран результатов и прогресса

## 5.1 Основные метрики
- Итог: `X из Y` правильных.
- Процент точности.
- Время прохождения (если трекается).
- Разбивка по темам (для режимов с несколькими вопросами).

## 5.2 Визуальные блоки
- Hero-метрика (крупная цифра процента).
- Стек/список метрик.
- Мини-граф прогресса (последние сессии, MVP можно как список последних результатов).
- CTA:
  - `Повторить`
  - `Попробовать другой режим`
  - `На главный`

## 5.3 Прогресс (глобально)
- Храним и показываем:
  - текущий стрик (опционально в MVP)
  - среднюю точность
  - количество решённых вопросов
- На главном экране: компактная версия прогресса.

## 6) Mobile-first адаптация

## 6.1 Брейкпоинты
- `xs`: 320–359
- `sm`: 360–413
- `md`: 414–767
- `lg`: 768+

## 6.2 Layout-правила
- Базовая ширина контента: 100%, внутренние отступы 16px (mobile).
- На `lg` ограничение контейнера: 720–840px по центру.
- Кнопки full-width на mobile, auto-width на tablet/desktop при необходимости.

## 6.3 Touch & ergonomics
- Минимальная зона тапа: **44x44px**.
- Основные CTA в нижней зоне экрана (thumb-friendly).
- Между интерактивными элементами: минимум 8px.

## 6.4 Performance
- Lazy-load изображений вопросов.
- Skeleton вместо layout-shift.
- Целевой LCP для мобильного: <2.5s на типичном 4G.

## 9) Accessibility (минимум для MVP)
- Контраст текста к фону: не ниже WCAG AA.
- Полная клавиатурная навигация на desktop.
- Visible focus для всех интерактивных элементов.
- ARIA-лейблы для кнопок без текста.
- Изображения вопросов с alt.
- Не полагаться только на цвет (добавлять иконку/текст в статусах).

## 11) Сценарии пользователя (MVP)

1. **Быстрый запуск (10–20 сек):**
   Главный → Случайный вопрос → Ответ → Мини-результат.

2. **Учёба по теме (3–5 мин):**
   Главный → По теме → Выбор темы → Серия вопросов → Результаты.

3. **Имитация билета:**
   Главный → Случайный билет → Прохождение N вопросов → Итог.

---

## 12) Definition of Done для UX MVP
- Реализованы 3 ключевых экрана.
- На главном есть 3 режима, каждый стартует корректный поток.
- Экран вопроса поддерживает текст + опциональную картинку + 4 ответа.
- Есть корректные состояния (loading/error/answered).
- Экран результатов показывает минимум: score, %, CTA.
- UI адаптирован под mobile-first и не ломается на 320px.
- Базовые дизайн-токены и компонентная библиотека зафиксированы.

---

## 13) Рекомендации для следующей итерации (post-MVP)
- Персональные рекомендации по слабым темам.
- Режим «ошибки недели» / повтор проблемных вопросов.
- Анимации микроинтеракций (motion tokens).
- Тёмная тема.
- Онбординг (1 экран) для новых пользователей.
